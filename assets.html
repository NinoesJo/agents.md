Project file map

How this AGENTS.md site fits together

This page explains the relationship between the files in this project: where the app starts, how the landing page is assembled, where styling comes from, and how static assets support the UI.

Big picture

  • pages defines the app shell and route.
  • components builds the homepage sections.
  • styles sets shared visual rules.
  • public stores logos and social images.

Relationship Map

Read this left to right: configuration and app shell feed into the route, the route composes sections, and those sections use shared styles plus public files.

Setup

package.json

Declares the Next.js, React, Tailwind, TypeScript, and Vercel Analytics dependencies. Its scripts run development and start commands.

Config

next.config.ts

Enables React strict mode and gives Next.js its project-level settings.

Shell

pages/_app.tsx

Wraps every page, loads global CSS, sets metadata, and adds Vercel Analytics.

Document

pages/_document.tsx

Controls the outer HTML document, including language, favicon links, and the body container.

Route

pages/index.tsx

Builds the homepage from section components and fetches GitHub contributor data for the example repository cards.

Sections

components/*.tsx

Provides the visible pieces of the page: hero, why section, compatibility logos, examples, how-to content, FAQ, and footer.

Styling

styles/globals.css

Imports Tailwind, defines fonts and theme colors, sets body styles, and contains the logo marquee animation.

Assets

public/

Holds browser-served files such as agent logos, favicons, and Open Graph images used by the homepage and social previews.

Main Components

The landing page is assembled from small, focused React components. Each component owns one idea on the page.

Hero.tsx Introduces AGENTS.md and shows a compact sample instructions file.
WhySection.tsx Explains why AGENTS.md exists alongside README.md.
CompatibilitySection.tsx Displays compatible coding agents and tools using logos from public/logos.
ExamplesSection.tsx Combines the sample markdown block with real repository cards.
ExampleListSection.tsx Renders repository cards and receives contributor avatars from pages/index.tsx.
CodeExample.tsx Displays Markdown-like examples and provides copy-to-clipboard behavior.
HowToUseSection.tsx Lists the practical steps for creating an AGENTS.md file.
FAQSection.tsx Answers common questions and shows configuration snippets.

How Data Moves

Most content is static React markup. The one dynamic piece is the contributor information used by the examples section.

Repos are listed

pages/index.tsx keeps a list of example GitHub repositories.

GitHub is queried

getStaticProps fetches contributor avatars and totals, with optional GH_AUTH_TOKEN support.

Props are passed

Contributor data is passed into ExamplesSection and then into ExampleListSection.

Cards render

Each card combines repo metadata, language color, avatars, and a link to that repo's AGENTS.md file.

This file is a standalone guide. The actual application remains the Next.js site defined by pages, components, styles, and public.
